# Security Policy\n\n## Reporting Vulnerabilities\n\nPlease report security vulnerabilities to: **security@example.com**\n\nDo NOT create public GitHub issues for security vulnerabilities.\n\n## Security Best Practices\n\n### For Users\n- Keep Node.js updated\n- Use strong API keys\n- Never share .env files\n- Use HTTPS in production\n- Regularly update dependencies\n\n### For Developers\n- Never commit secrets\n- Validate all inputs\n- Use parameterized queries\n- Sanitize outputs\n- Keep dependencies updated\n- Use security headers\n- Enable CORS restrictions\n- Log security events\n\n## Supported Versions\n\n| Version | Status | Support Until |\n|---------|--------|---------------|\n| 1.0.x   | Active | 2027-07-21   |\n| 0.x     | EOL    | 2026-07-21   |\n\n## Security Headers\n\nRecommended headers for production:\n```\nStrict-Transport-Security: max-age=31536000\nX-Content-Type-Options: nosniff\nX-Frame-Options: DENY\nX-XSS-Protection: 1; mode=block\nContent-Security-Policy: default-src 'self'\n```\n\n## Dependency Audit\n\n```bash\n# Check for vulnerabilities\nnpm audit\n\n# Fix vulnerabilities\nnpm audit fix\n```\n\n## Known Security Issues\n\nNone currently reported.\n\n## Security Update Timeline\n\n- Critical: Fixed within 24 hours\n- High: Fixed within 1 week\n- Medium: Fixed within 1 month\n- Low: Fixed in next release\n\n---\n\n**Last Updated:** 2026-07-21\n
