# FAH Setup Guide\n\n## Quick Start (5 minutes)\n\n### Prerequisites\n- Node.js 14+ ([Download](https://nodejs.org/))\n- npm 6+ (comes with Node.js)\n- Git\n\n### Installation\n\n```bash\n# Clone repository\ngit clone https://github.com/Anmol2046S/FAH.git\ncd FAH\n\n# Install dependencies\nnpm install\n\n# Setup environment\ncp .env.example .env\n\n# Start server\nnpm start\n\n# Open browser\nhttp://localhost:3000\n```\n\n## Detailed Setup\n\n### Step 1: Install Node.js\n\n#### Windows\n1. Download from https://nodejs.org/\n2. Run the installer\n3. Follow the wizard\n4. Verify: `node --version` and `npm --version`\n\n#### Mac\n```bash\n# Using Homebrew\nbrew install node\n\n# Verify\nnode --version\nnpm --version\n```\n\n#### Linux\n```bash\n# Ubuntu/Debian\nsudo apt-get update\nsudo apt-get install nodejs npm\n\n# Verify\nnode --version\nnpm --version\n```\n\n### Step 2: Clone the Repository\n\n```bash\ngit clone https://github.com/Anmol2046S/FAH.git\ncd FAH\n```\n\n### Step 3: Install Dependencies\n\n```bash\nnpm install\n\n# Or if you prefer yarn\nyarn install\n```\n\nThis reads `package.json` and installs all required packages.\n\n### Step 4: Configure Environment\n\n1. Copy the example file:\n   ```bash\n   cp .env.example .env\n   ```\n\n2. Edit `.env` with your settings:\n   ```env\n   PORT=3000\n   NODE_ENV=development\n   API_KEY=your_ai_api_key_here\n   LOG_LEVEL=info\n   ```\n\n### Step 5: Start the Application\n\n```bash\nnpm start\n```\n\nYou should see:\n```\n✓ Server running on http://localhost:3000\n✓ API ready\n```\n\n### Step 6: Verify Installation\n\nOpen browser: `http://localhost:3000`\n\nYou should see the FAH application interface.\n\n## Docker Setup\n\n### Prerequisites\n- Docker installed\n- Docker Compose (optional)\n\n### Build and Run\n\n```bash\n# Build image\ndocker build -t fah-app .\n\n# Run container\ndocker run -p 3000:3000 --env-file .env fah-app\n\n# Access at http://localhost:3000\n```\n\n### Using Docker Compose\n\nIf `docker-compose.yml` exists:\n\n```bash\ndocker-compose up --build\n```\n\n## Development Mode\n\nFor automatic restart on file changes:\n\n```bash\nnpm run dev\n```\n\nRequires `nodemon` to be installed.\n\n## Troubleshooting\n\n### Issue: \"npm: command not found\"\n**Solution:** Node.js not installed. Download from https://nodejs.org/\n\n### Issue: \"Port 3000 already in use\"\n**Solution:**\n```bash\n# Use different port\nPORT=3001 npm start\n\n# Or kill process on port 3000\n# Windows: netstat -ano | findstr :3000\n# Linux/Mac: lsof -i :3000\n```\n\n### Issue: \"Cannot find module\"\n**Solution:**\n```bash\n# Reinstall dependencies\nrm -rf node_modules package-lock.json\nnpm install\n```\n\n### Issue: \"API key invalid\"\n**Solution:**\n- Verify API key in `.env`\n- Check key has proper permissions\n- Regenerate key if necessary\n\n### Issue: Docker build fails\n**Solution:**\n```bash\n# Clear cache and rebuild\ndocker system prune\ndocker build --no-cache -t fah-app .\n```\n\n## Environment Variables Explained\n\n```env\nPORT=3000                          # Server port\nNODE_ENV=development               # development/production\nAPI_KEY=your_ai_api_key            # AI service key\nLOG_LEVEL=info                     # Logging level\nHEALTH_CHECK_INTERVAL=30000        # Health check frequency (ms)\n```\n\n## Common Commands\n\n```bash\n# Start application\nnpm start\n\n# Start with auto-reload (development)\nnpm run dev\n\n# Run tests\nnpm test\n\n# Run linter\nnpm run lint\n\n# Build for production\nnpm run build\n\n# Check Node version\nnode --version\n\n# Check npm version\nnpm --version\n\n# Update npm\nnpm install -g npm@latest\n```\n\n## Project Structure\n\n```\nFAH/\n├── index.html          # Frontend UI\n├── index.js            # Backend server\n├── package.json        # Dependencies\n├── .env                # Environment variables (not in git)\n├── .env.example        # Template for .env\n├── Dockerfile          # Docker configuration\n├── .dockerignore        # Docker ignore patterns\n├── node_modules/       # Installed packages\n└── README.md           # Documentation\n```\n\n## Next Steps\n\n1.
